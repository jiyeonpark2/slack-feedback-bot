## 1. 봇 소개 및 사용 방법

### 이 봇은 왜 만들었나요?

배너 피드백 작업에 리소스가 많이 소요되어, AI를 통해 이 과정을 효율화하기 위해 만들었습니다.  
이 봇은 **피그마 링크를 보내는 것만으로 Claude AI가 배너를 자동 분석하고 피드백을 제공해 주는** 도구입니다.

### 봇이 하는 일

봇을 멘션(`@봇이름`)하고 `/피드백` 명령어와 함께 피그마 링크를 보내면, Claude AI가 해당 링크의 배너들을 분석해 스레드 댓글로 피드백을 자동 전송합니다.

| 기능 | 설명 |
|------|------|
| 배너 분석 요청 | 봇 멘션 + `/피드백` + 피그마 링크를 입력하면 분석이 시작됩니다 |
| AI 자동 피드백 | Claude AI가 배너 디자인을 분석하고 피드백을 스레드 댓글로 달아줍니다 |
| 다항목 분석 | 시각적 계층, 타이포그래피, 색상, 레이아웃, 개선 제안 등을 한번에 제공합니다 |

### 봇이 실행되고 있는지 확인하는 방법

Railway 대시보드 또는 슬랙에서 봇에게 말을 걸어 반응이 오면 실행 중입니다.  
반응이 없으면 [Railway 배포 관리 방법](#5-railway-배포-관리-방법) 섹션을 참고하세요.

## 2. 슬랙에서 명령어 사용법

### 방법 1: 봇 멘션 + `/피드백` 명령어로 배너 분석 요청하기

채널에서 `@봇이름`을 태그하고 `/피드백` 명령어와 피그마 링크를 함께 입력합니다.

```
@피드백봇 /피드백 https://www.figma.com/file/XXXXXX/배너파일?node-id=0%3A1
```

봇이 스레드에 이렇게 분석 결과를 달아줍니다:

```
🎨 배너 분석 결과입니다!

📐 시각적 계층 구조: 헤드라인과 CTA 버튼의 크기 대비가 명확합니다.
✍️ 타이포그래피: 본문 폰트 크기가 작아 모바일에서 가독성이 낮을 수 있습니다.
🎨 색상 및 대비: 배경과 텍스트의 대비율이 기준치를 충족합니다.
📏 레이아웃: 좌측 여백이 우측보다 좁아 시각적 불균형이 있습니다.

💡 개선 제안:
1. 본문 폰트를 14px → 16px로 키우면 가독성이 향상됩니다.
2. 좌우 여백을 동일하게 맞춰 균형감을 높여보세요.
```

### 방법 2: 봇에게 DM으로 피드백 보내기

공개하기 민감한 내용은 봇에게 직접 메시지를 보내도 됩니다.  
슬랙 좌측 메뉴에서 봇을 찾아 DM을 보내면 동일하게 처리됩니다.

### 방법 3: `/피드백` 명령어로 피그마 분석하기

피그마 디자인 링크와 함께 `/피드백` 명령어를 사용합니다.

```
/피드백 https://www.figma.com/file/XXXXXX/파일이름?node-id=0%3A1
```

봇이 다음 내용을 분석해서 채널에 올려줍니다:
- 시각적 계층 구조 (정보 우선순위, 시선 흐름)
- 타이포그래피 (폰트, 크기, 가독성)
- 색상 및 대비
- 레이아웃 균형 (여백, 요소 배치)
- 개선 제안 2~3가지

> **주의**: 피그마 분석 기능을 쓰려면 Anthropic API 키와 Figma 액세스 토큰이 설정되어 있어야 합니다. 설정되지 않았다면 [3번 섹션](#3-토큰-및-api-키-관리-방법)을 참고하세요.

## 3. 토큰 및 API 키 관리 방법

봇이 동작하려면 외부 서비스와 연결하기 위한 "비밀번호" 같은 값들이 필요합니다.  
이 값들은 **절대로 코드에 직접 쓰거나 카카오톡/이메일로 공유하면 안 됩니다.**

### 필요한 키 목록

| 키 이름 | 용도 | 어디서 발급? |
|--------|------|------------|
| `SLACK_BOT_TOKEN` | 슬랙 봇 인증 (`xoxb-`로 시작) | [Slack API 사이트](https://api.slack.com/apps) → 앱 선택 → OAuth & Permissions |
| `SLACK_APP_TOKEN` | 슬랙 실시간 연결 (`xapp-`로 시작) | Slack API 사이트 → Basic Information → App-Level Tokens |
| `ANTHROPIC_API_KEY` | Claude AI 사용 (`sk-ant-`로 시작) | [Anthropic 콘솔](https://console.anthropic.com) → Settings → API Keys |
| `FIGMA_ACCESS_TOKEN` | 피그마 이미지 가져오기 (`figd_`로 시작) | 피그마 앱 → 우측 상단 프로필 → Settings → Security → Personal access tokens |
| `FEEDBACK_CHANNEL_ID` | 피드백 자동 전달할 채널 ID (`C`로 시작) | 슬랙에서 채널 우클릭 → "채널 ID 복사" |

### Railway에서 키 설정하는 방법

Railway에서 배포 중이라면, 코드가 아닌 Railway 대시보드에서 키를 관리합니다.

1. [Railway 대시보드](https://railway.app)에 로그인합니다.
2. 해당 프로젝트를 클릭합니다.
3. 서비스 카드를 클릭한 뒤 상단 탭에서 **Variables**를 선택합니다.
4. `+ New Variable` 버튼을 클릭합니다.
5. 키 이름(예: `ANTHROPIC_API_KEY`)과 값을 입력하고 저장합니다.
6. 저장 후 봇이 자동으로 재시작됩니다.

### 로컬(내 컴퓨터)에서 키 설정하는 방법

프로젝트 폴더에 `.env` 파일이 있습니다. 이 파일을 메모장이나 텍스트 에디터로 열어 수정합니다.

```
SLACK_BOT_TOKEN=xoxb-여기에-토큰-입력
SLACK_APP_TOKEN=xapp-여기에-토큰-입력
ANTHROPIC_API_KEY=sk-ant-여기에-키-입력
FIGMA_ACCESS_TOKEN=figd_여기에-토큰-입력
FEEDBACK_CHANNEL_ID=C여기에-채널ID
```

> `.env` 파일은 깃허브에 올라가지 않습니다. 파일을 잃어버리면 각 서비스에서 새로 발급해야 합니다.

### 키가 만료되거나 문제가 생겼을 때

- **슬랙 토큰**: 앱이 재설치되거나 권한이 변경되면 새로 발급됩니다
- **Anthropic 키**: console.anthropic.com에서 기존 키 삭제 후 새 키 생성
- **Figma 토큰**: 피그마 설정에서 기존 토큰 삭제 후 새로 생성
- 새 키 발급 후 Railway Variables에 업데이트하면 자동으로 적용됩니다

## 4. 코드 수정하는 방법

### 코드는 어디에 있나요?

핵심 코드는 `app.js` 파일 하나에 모여 있습니다.

```
slack-feedback-bot/
├── app.js          ← 봇의 모든 동작이 여기에 있습니다
├── package.json    ← 프로그램 설정 파일 (손대지 않아도 됩니다)
├── .env            ← API 키 보관 파일 (깃허브에 올리면 안 됩니다!)
└── .env.example    ← .env 파일 예시 (참고용)
```

### 자주 수정할 수 있는 부분

#### 피드백 접수 메시지 문구 바꾸기

`app.js` 파일에서 아래처럼 생긴 부분을 찾으면 됩니다. 큰따옴표(`"`) 안의 텍스트를 원하는 내용으로 바꾸세요.

```javascript
// 예: "✅ 피드백이 접수되었습니다!" 를 다른 문구로 바꾸고 싶다면
// 이 부분을 수정합니다
text: "✅ 피드백이 접수되었습니다!"
```

#### 피그마 분석 항목 바꾸기

`app.js`에서 `분석해주세요` 또는 `프롬프트`라고 적힌 부분 근처에 Claude에게 보내는 지시문이 있습니다.  
분석 항목을 추가하거나 빼고 싶다면 그 텍스트를 수정하면 됩니다.

### 수정 절차 (깃허브 사용 시)

1. 깃허브에서 `app.js` 파일을 엽니다.
2. 연필 아이콘(✏️)을 눌러 편집 모드로 전환합니다.
3. 원하는 내용을 수정합니다.
4. 하단 **Commit changes** 버튼을 눌러 저장합니다.
5. Railway가 깃허브와 연결되어 있다면 자동으로 배포됩니다.

### 수정 절차 (내 컴퓨터에서 직접 수정 시)

1. 프로젝트 폴더에서 `app.js`를 메모장이나 VS Code로 엽니다.
2. 원하는 내용을 수정하고 저장합니다.
3. 터미널(명령 프롬프트)을 열고 프로젝트 폴더로 이동합니다.
4. 아래 명령어로 깃허브에 올립니다:
   ```
   git add .
   git commit -m "수정 내용 간단 설명"
   git push
   ```
5. Railway에서 자동으로 새 버전을 배포합니다.

> **팁**: 코드 수정이 어렵다면 Claude.ai나 ChatGPT에 `app.js` 내용을 붙여넣고 "이 부분을 이렇게 바꿔줘"라고 부탁하면 됩니다.

## 5. Railway 배포 관리 방법

Railway는 봇을 24시간 클라우드에서 돌려주는 서비스입니다.  
내 컴퓨터를 켜지 않아도 봇이 항상 작동합니다.

### Railway 대시보드 접속

1. [railway.app](https://railway.app)에 접속합니다.
2. 깃허브 계정으로 로그인합니다 (봇 제작자 계정으로 초대받아야 접근 가능합니다).
3. 프로젝트 목록에서 `slack-feedback-bot`을 클릭합니다.

### 봇 상태 확인하기

대시보드에서 서비스 카드를 클릭하면:
- **Deployments** 탭: 최근 배포 기록과 현재 상태를 볼 수 있습니다.
  - 초록색 체크 = 정상 실행 중
  - 빨간색 X = 오류로 중단됨
- **Logs** 탭: 봇의 실시간 로그(작동 기록)를 볼 수 있습니다.

### 봇이 멈췄을 때 재시작하기

1. Railway 대시보드에서 서비스를 클릭합니다.
2. **Deployments** 탭에서 가장 최근 배포를 클릭합니다.
3. 우측 상단 **Redeploy** 버튼을 클릭합니다.
4. 1~2분 후 봇이 다시 실행됩니다.

### 오류 원인 파악하기

봇이 멈췄다면 **Logs** 탭에서 빨간색 오류 메시지를 찾습니다.  
자주 발생하는 오류와 해결 방법:

| 오류 메시지 | 원인 | 해결 방법 |
|------------|------|----------|
| `Invalid token` | Slack 토큰이 잘못되거나 만료됨 | Slack에서 새 토큰 발급 후 Variables 업데이트 |
| `ANTHROPIC_API_KEY is not set` | Anthropic 키가 없음 | Variables에 키 추가 |
| `Figma API error` | Figma 토큰 문제 또는 잘못된 URL | Figma 토큰 확인, URL 형식 확인 |
| `Cannot find module` | 패키지 설치 오류 | Redeploy 버튼으로 재배포 |

### 요금 관련

Railway는 무료 플랜(Hobby)과 유료 플랜(Pro)이 있습니다.
- 무료 플랜: 월 $5 크레딧 제공, 소규모 봇 운영에 충분
- 크레딧이 소진되면 봇이 자동으로 멈춥니다
- 대시보드 상단에서 남은 크레딧 확인 가능
