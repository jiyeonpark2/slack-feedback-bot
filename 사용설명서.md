# 피드백봇 사용설명서

> 피그마 배너 링크만 보내면 Claude AI가 알아서 디자인 피드백을 달아주는 슬랙 봇입니다.

---

## 언제 쓰면 좋나요?

- 배너 시안이 나왔는데 디자인 피드백을 빠르게 받고 싶을 때
- 여러 배너를 한꺼번에 검토해야 할 때
- 매번 같은 피드백을 반복하는 게 번거로울 때

---

## 슬랙에서 쓰는 방법

딱 한 가지만 기억하면 됩니다.

**① 봇을 멘션하고 ② `/피드백`을 입력하고 ③ 피그마 링크를 붙여넣기**

```
@피드백봇 /피드백 https://www.figma.com/file/여기에-링크-붙여넣기
```

잠시 후 봇이 스레드에 이런 식으로 피드백을 달아줍니다:

```
🎨 배너 분석 결과입니다!

1️⃣ 오탈자/문법 — ✅ OK
2️⃣ 제품명 유무 — ✅ OK
3️⃣ 할인 명분 유무 — ⚠️ 개선 필요 (왜 할인하는지 이유가 없어요. 예: "여름 특가", "창립 기념" 등 명분을 추가하면 신뢰도가 올라가요.)
   [ 짧은 대안 ]  [ 자유 대안 ]
4️⃣ 가격 혜택 유무 — ✅ OK
5️⃣ 기간 한정성 유무 — ⚠️ 개선 필요 (마감 기한이 보이지 않아요. "~7/31까지" 같은 문구로 긴박감을 줄 수 있어요.)
   [ 짧은 대안 ]  [ 자유 대안 ]
6️⃣ 물량 한정성 유무 — ✅ OK
```

버튼을 누르면:
- **짧은 대안**: 바로 쓸 수 있는 문구 2~3개를 추천해줍니다. (예: "여름 특가", "앱 출시 기념", "단독 한정")
- **자유 대안**: 원하는 방향을 직접 입력하면 그에 맞는 문구를 만들어줍니다.

> **피그마 링크 복사 방법**: 피그마에서 분석할 프레임을 선택 → 우클릭 → "Copy link to selection"

## 이 파일이 있으면 누구나 봇을 쓰고, 고치고, 발전시킬 수 있어요

이 사용설명서와 프로젝트의 `app.js` 파일을 **Claude AI (claude.ai)** 에 붙여넣고 원하는 걸 말하면 됩니다.

예를 들어:
- "피드백 항목에 '모바일/PC 대응 여부'도 추가해줘"
- "분석 결과를 더 짧고 간결하게 바꿔줘"
- "봇이 오류가 나는데 왜 그런지 봐줘"

개발 지식이 없어도 괜찮습니다. Claude AI가 코드를 읽고 수정까지 도와줍니다.

## 봇 구조 및 코드 위치

### 파일 구조

```
slack-feedback-bot/
├── app.js          ← 봇의 모든 동작이 담긴 핵심 파일 (여기만 수정하면 됩니다)
├── package.json    ← 사용하는 라이브러리 목록 (건드리지 않아도 됩니다)
├── .env            ← API 키 보관 파일 (깃허브에 올리면 안 됩니다!)
└── .env.example    ← .env 파일 양식 예시
```

**수정이 필요하다면 `app.js` 파일 하나만 보면 됩니다.**  
슬랙 명령어 처리, 피그마 이미지 추출, Claude AI 분석 요청, 결과 전송이 모두 이 파일 안에 있습니다.

### 봇이 작동하는 순서

```
사용자가 슬랙에서 명령어 입력
    ↓
봇이 피그마 링크에서 배너 이미지 추출 (Figma API)
    ↓
이미지를 Claude AI에게 전달 (Anthropic API)
    ↓
Claude AI가 분석 후 피드백 텍스트 생성
    ↓
봇이 슬랙 스레드에 피드백 댓글 게시
```

### 필요한 API 키 (`.env` 파일 또는 Railway Variables에 보관)

| 키 이름 | 용도 |
|--------|------|
| `SLACK_BOT_TOKEN` | 슬랙 봇 인증 |
| `SLACK_APP_TOKEN` | 슬랙 실시간 연결 |
| `ANTHROPIC_API_KEY` | Claude AI 호출 |
| `FIGMA_ACCESS_TOKEN` | 피그마 이미지 추출 |

## GitHub 레포지토리

코드 원본은 깃허브에 있습니다.

**👉 https://github.com/jiyeonpark2/slack-feedback-bot**

### 코드를 수정하고 배포하는 방법

1. 위 주소에서 `app.js` 파일을 클릭합니다.
2. 우측 상단 연필 아이콘(✏️)을 클릭해 편집 모드로 전환합니다.
3. 원하는 내용을 수정합니다.
4. 하단 **Commit changes** 버튼을 눌러 저장합니다.
5. Railway가 자동으로 감지해 새 버전을 배포합니다 (1~2분 소요).

> 코드 수정이 막막하다면 `app.js` 전체 내용을 복사해서 Claude AI에게 붙여넣고 "이렇게 바꿔줘"라고 하면 됩니다.

## Railway 배포 관리

봇은 **Railway** 라는 클라우드 서비스에서 24시간 실행됩니다. 내 컴퓨터를 켜두지 않아도 됩니다.

### 대시보드 접속

👉 https://railway.app → 깃허브 계정으로 로그인 → `slack-feedback-bot` 프로젝트 클릭

### 봇 상태 확인

서비스 카드 클릭 후:
- **Deployments 탭**: 초록색 체크 = 정상, 빨간색 X = 오류
- **Logs 탭**: 실시간 작동 기록 확인 가능

### 봇이 멈췄을 때

1. Deployments 탭 → 최근 배포 클릭
2. **Redeploy** 버튼 클릭
3. 1~2분 후 재시작 완료

### API 키(환경변수) 변경할 때

1. 서비스 카드 클릭 → **Variables 탭** 선택
2. 변경할 키를 찾아 값 수정 후 저장
3. 봇 자동 재시작됨

### Railway 요금

- 무료 플랜: 월 $5 크레딧 제공 (소규모 운영에 충분)
- 크레딧 소진 시 봇 자동 정지 → 대시보드에서 잔액 확인 후 충전

## 주의사항

**Anthropic API 크레딧이 있어야 봇이 작동합니다.**

크레딧이 다 떨어지면 봇이 피드백을 보내지 않습니다 (에러 없이 조용히 멈추는 경우도 있어요).

충전 방법:
1. [console.anthropic.com](https://console.anthropic.com) 접속
2. 로그인 후 **Billing** 메뉴 클릭
3. **Add credits** 버튼으로 충전 ($5부터 가능)

> 소규모로 사용할 경우 $5으로 꽤 오래 쓸 수 있습니다. 잔액은 같은 페이지에서 확인할 수 있어요.
